Uploading PDF Documents

PDF documents can be uploaded individually or in bulk, with or without using configured Upload folders.

Configured Upload Folders are used to automate the uploading of PDF documents to the database. For each PDF document that is created in or copied into a configured upload folder will have PDF Information Properties defined for that folder written to a clone of the PDF file; and then the PDF file clone is uploaded to the database during the next upload cycle.

Each document is stored in the database as follows:

ID- Assigned to the document record by the database.

Title - Extracted from the PDF Information Properties. This column is searchable.

Author - Extracted from the PDF Information Properties. This column is searchable.

Subject - Extracted from the PDF Information Properties. This column is searchable.

Keywords - Extracted from the PDF Information Properties. This column is searchable.

Added - Date and time the document was added to the database. This column is searchable.

Notes - User editable notes. This column is search able.

PDF - The text content of the PDF document. This column is search able.

Additional Information:

  • USER password protected PDF documents are not supported by PDFKeeper. If you have the legal right to remove the password, you can find plenty of information on the Internet to assist you.

  • The upload process requires that all PDF documents contain an Title, Author, and Subject.

  • Oracle Database XE does not index annotations contained within a PDF document.

  • The upload process will check for PDF documents that need to be uploaded every five seconds.

  • It is NOT recommended to configure the Windows Recycle Bin to delete files immediately as the Upload process will delete all preprocessed and cloned PDF documents to the Recycle Bin.
