Uploading PDF Documents

PDF documents can be uploaded individually or in bulk, with or without using configured Upload folders to allow for integration and automation.

Configured Upload Folders are used to automate the uploading of PDF documents to the database. For each PDF document that is created in or copied into a configured upload folder, it will have PDF Metadata defined for that folder written to a clone of the PDF file; and then the PDF file clone is uploaded to the database during the next upload cycle.

Each document is stored in the database as follows:

ID – Assigned to the document record by the database.

Title – Extracted from the PDF Metadata. This column is searchable.

Author – Extracted from the PDF Metadata. This column is searchable.

Subject – Extracted from the PDF Metadata. This column is searchable.

Keywords – Extracted from the PDF Metadata. This column is searchable.

Added – Date and time the document record was added to the database. This column is searchable.

Notes – User editable notes. This column is searchable.

PDF – Content of the PDF document. This column is searchable except when using the local database option.

Category – Optional category of the document record. This column is searchable.

Flag – Flag document state. 0 = not flagged, 1 = flagged.

Tax Year – Optional tax year of the document record. This column is searchable.

Text Annotations – Extracted from the PDF. This column is searchable.

Text – Extracted from the PDF. This column is searchable.

Additional Information:

  • USER password protected PDF documents are not supported by PDFKeeper. If you have the legal right to remove the password, you can find plenty of information on the Internet to assist you.

  • The upload process requires that all PDF documents contain a Title, Author, and Subject.

  • Annotations in a PDF document are not filtered by Oracle Database.

  • Embedded fonts in a PDF document are not filtered correctly by Oracle Database.

  • The upload process will check for PDF documents that need to be uploaded every five seconds.

  • Each preprocessed PDF document in the Upload folder will be copied to <original name>-<GUID>.pdf and then renamed to <original name>.delete to avoid a file in use prompt during the upload cycle. The copied PDF document will be deleted to the Windows Recycle Bin after being processed.

  • It is NOT recommended to configure the Windows Recycle Bin to remove files immediately when deleted.

  • Before restoring preprocessed PDF documents from the Recycle Bin, make sure to close PDFKeeper. After restoring, move the file(s) out of the Upload folder to prevent PDFKeeper from uploading the PDF documents to the database after PDFKeeper has been restarted.
