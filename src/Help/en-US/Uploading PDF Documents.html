Uploading PDF Documents

PDF documents can be uploaded individually or in bulk, with or without using configured Upload folders to allow for integration and automation.

Configured Upload Folders are used to automate the uploading of PDF documents to the database. For each PDF document that is created in or copied into a configured upload folder, it will have PDF Information Properties defined for that folder written to a clone of the PDF file; and then the PDF file clone is uploaded to the database during the next upload cycle.

Each document is stored in the database as follows:

ID – Assigned to the document record by the database.

Title – Extracted from the PDF Information Properties. This column is searchable.

Author – Extracted from the PDF Information Properties. This column is searchable.

Subject – Extracted from the PDF Information Properties. This column is searchable.

Keywords – Extracted from the PDF Information Properties. This column is searchable.

Added – Date and time the document was added to the database. This column is searchable.

Notes – User editable notes. This column is searchable.

PDF – The text content of the PDF document. This column is searchable.

Category – The optional category of the PDF. This column is searchable.

Flag – Flag document state. 0 = not flagged, 1 = flagged.

Additional Information:

  • USER password protected PDF documents are not supported by PDFKeeper. If you have the legal right to remove the password, you can find plenty of information on the Internet to assist you.

  • The upload process requires that all PDF documents contain a Title, Author, and Subject.

  • Annotations in a PDF document are not filtered by Oracle Database.

  • Embedded fonts in a PDF document are not filtered correctly by Oracle Database.

  • The upload process will check for PDF documents that need to be uploaded every five seconds.

  • It is NOT recommended to configure the Windows Recycle Bin to delete files immediately as the Upload process will delete preprocessed PDF documents to the Recycle Bin.

  • Before restoring preprocessed PDF documents from the Recycle Bin, make sure to close PDFKeeper. After restoring, move the file(s) out of the Upload folder to prevent PDFKeeper from uploading the PDF documents to the database after PDFKeeper has been restarted.
