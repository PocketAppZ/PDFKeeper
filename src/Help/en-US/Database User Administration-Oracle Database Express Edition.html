Database User Administration

Oracle Database Express Edition (XE)

It is recommended that each user be assigned a personal database account, with only the required privileges. The pdfkeeper database account should only be used for pdfkeeper schema administration, not for general use. For more information on Managing Users, please refer to the 2 Day DBA book in the Oracle Database Express Edition Documentation.

Perform the following steps for any user that will need access to the pdfkeeper schema:

  1. From a Command Prompt, execute sqlplus

  2. When logged onto the computer that has Oracle Database XE installed and using the account that installed Oracle Database XE, enter connect / as sysdba. Otherwise, enter the database connect string in the format username@host:port where username is a database account that must be a member of SYSDBA (SYSTEM is a member of SYSDBA); host is the host name or IP address of the computer that is running Oracle Database XE (use localhost or 127.0.0.1 if logged onto the same computer as Oracle Database XE); and port (optional) is the TCP port number on which the Oracle Net listener is listening (the default port number is 1521).

  3. Enter create user user_name default tablespace users identified by "password";

  4. Enter grant connect, resource to user_name;

  5. Enter quit

  6. From a Command Prompt, execute sqlplus

  7. When logged onto the computer that has Oracle Database XE installed and using the account that installed Oracle Database XE, enter connect pdfkeeper. Otherwise, enter the database connect string in the format pdfkeeper@host:port where host is the host name or IP address of the computer that is running Oracle Database XE (use localhost or 127.0.0.1 if logged onto the same computer as Oracle Database XE); and port (optional) is the TCP port number on which the Oracle Net listener is listening (the default port number is 1521).

  8. For a new account, enter the following statement to grant the user select/query access: grant select on docs to user_name;

  9. To give the user the ability to upload documents, enter the following statements: grant insert on docs to user_name; and grant select on docs_seq to user_name;

  10. To give the user the ability to edit document notes, enter the following statement: grant update on docs to user_name;

  11. To give the user the ability to delete documents, enter the following statement: grant delete on docs to user_name;

  12. To prevent the user from uploading documents, enter the following statements: revoke insert on docs from user_name; and revoke select on docs_seq from user_name;

  13. To prevent the user from editing document notes, enter the following statement: revoke update on docs from user_name;

  14. To prevent the user from deleting documents, enter the following statement: revoke delete on docs from user_name;
