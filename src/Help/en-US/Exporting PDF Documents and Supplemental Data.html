Exporting PDF Documents and Supplemental Data

To export the PDF document and corresponding supplemental data from one or more document records returned from a search, perform the following steps:

  1. On the Search Results view, click on the check box for each record that you wish to export. To select all documents in the Search Results view, select the File menu, then Select, and then All; or press Alt + F, then L, and then A on the keyboard.

  2. Select the File menu, and then select Export; or press Alt + F, and then T on the keyboard.

  3. Select the folder location where to create the export folder that will contain the exported PDF documents and corresponding supplemental data files.

  4. Select OK or press Enter on the keyboard to perform the export.

Additional Information:

  • You can navigate the Search Results view using the up and down arrows on the keyboard and check/uncheck a document record using the space bar on the keyboard.

  • To uncheck all document records in the Search Results view, select the File menu, then Select, and then None; or press Alt + F, then L, and then N on the keyboard.

  • The absolute path name of the folder where the exported files are located will be displayed in a message box after the export has completed.

  • Exporting is a convenient way to transfer PDF documents and supplemental data to another database system without having to use the source and target database systems export/import processes. The export will create a folder that contains each exported PDF document named [<ID>]<Title>.PDF and corresponding supplemental data file [<ID>]<Title>.XML that contains the Notes, Category, and Flag state stored in the database for that document. All exported files will be organized in sub-folders by Author and Subject.

  • When the Title, Author, Subject, and Keywords in the metadata of the exported PDF does not match the Title, Author, Subject, and Keywords in the database document record, the PDF will be rewritten with the Title, Author, Subject, and Keywords from the database document record.

  • When exporting documents for transfer to another system, it is highly recommended that nobody else be logged into PDFKeeper.
