Database Setup

Oracle Database Express Edition (XE)

This topic provides the steps that need to be performed for first time setup of Oracle Database XE for use with PDFKeeper.

For an existing database schema that was built with a setup script from any version of PDFKeeper starting with 2.0.0 and ending with 3.1.2 or had to change from the WORLD_LEXER to the BASIC_LEXER due to the WORLD_LEXER bug in Oracle Database XE 10g can make the switch back to the WORLD_LEXER if so desired by starting with Step 4; however, Oracle Database XE must be at a minimum of 11g to avoid the bug.

The WORLD_LEXER offers broader language support and is now the lexer that is set when the PDFKeeper database schema is created.

  1. Please read the Oracle Database Express Edition Documentation for instructions on how to install, configure, and administer an Oracle Express Edition Database.

  2. Install Oracle Database Express Edition on the system that will host the database.

  3. If Oracle Database XE was installed on a different computer, you will need to download Oracle Instant Client Basic or Basic Lite and SQL*Plus packages to this computer, extract the ZIP files into an empty folder, and then add that folder to the PATH.

  4. For first time setup, execute Oracle Database Schema Setup; to switch an existing schema from BASIC_LEXER to WORLD_LEXER, execute Oracle Database Basic To World Lexer Switch

  5. From a Command Prompt, execute sqlplus

  6. When logged onto the computer that has Oracle Database XE installed and using the account that installed Oracle Database XE, enter connect / as sysdba. Otherwise, enter the database connect string in the format username@host:port where username is a database account that must be a member of SYSDBA (SYSTEM is a member of SYSDBA); host is the host name or IP address of the computer that is running Oracle Database XE (use localhost or 127.0.0.1 if logged onto the same computer as Oracle Database XE); and port (optional) is the TCP port number on which the Oracle Net listener is listening (the default port number is 1521).

  7. Change the pdfkeeper database user account password by entering alter user pdfkeeper identified by "new_password";
